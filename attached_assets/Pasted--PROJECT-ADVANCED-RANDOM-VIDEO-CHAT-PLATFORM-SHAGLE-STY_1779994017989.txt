🧠 MATCHING LOGIC (BACKEND)

When matching users:

Step 1: collect users in queue

Each queued user includes:

{
  socketId,
  gender,
  country,
  interests: ["music", "gaming"]
}
Step 2: filter matching rules

Match only if:

Gender rule:
If user selected “Female only” → match only females
If “Male only” → match only males
Else → all genders
Country rule:
“Same country” → must match country
Else → ignore
Interest rule (optional):

Calculate similarity score:

+1 point per shared keyword
Prefer highest score match

Example:

score = sharedInterests.length

Match highest score available.

💬 SOCKET.IO CHAT FLOW (UPDATED)

When user sends message:

Step 1: send message to server
{
  from,
  to,
  message,
  originalLang
}
Step 2: backend detects receiver language
receiverLang = user.language
Step 3: translate message

Call LibreTranslate:

POST https://libretranslate.de/translate
{
  q: message,
  source: originalLang || "auto",
  target: receiverLang,
  format: "text"
}
Step 4: emit translated message
socket.emit("chat_message", {
  translatedText,
  originalText,
  from
});
